# HotPulse AI热点雷达需求方案设计文档

## 1. 需求功能详细设计

### 1.1 功能模块划分

**核心模块：**
- **用户管理模块**：注册、登录（无需权限管理，每个用户只能使用自己的关键词和热点）
- **关键词管理模块**：关键词输入、编辑、删除、停/启用、优先级设置、立即扫描功能
- **热点监控模块**：定时搜集、AI识别过滤、结果展示（过滤后的最终结果）
- **通知模块**：热点通知、通知设置、页面消息通知
- **Agent Skills模块**：核心功能封装（Web版验收后）

### 1.2 核心功能详细说明

**1. 关键词管理：**
- 支持手动输入、编辑、删除关键词
- 关键词优先级设置（高、中、低）
- **停/启用关键词**功能
- **立即扫描功能**：关键词添加成功后，用户可点击“立即扫描”按钮，马上开始搜索热点
- **简化管理**：不需要关键词分类管理和历史记录

**2. 定时热点搜集：**
- 可设置搜集频率（每小时、每天、自定义）
- **可设置搜集范围**：百度、必应、Twitter、B站、知乎、DuckDuckGo、HackerNews、Google
- 支持多源数据整合
- 自动去重和排序

**3. AI识别功能：**
- 集成OpenRouter AI服务
- 识别假冒内容、低质量内容、无关内容
- **具体过滤规则**：
  - 内容质量过滤：去除低质量、重复内容
  - 时效性过滤：内容时效超过48小时的不要
  - 互动量过滤：点赞低于50的不要，阅读量低于1.5k的不要
  - 来源可信度过滤：不可信来源过滤
- 内容相关性评分
- 智能过滤和排序

**4. 通知功能：**
- **异步即时通知**：符合条件的热点立即通知
- **页面消息通知**：在页面上显示通知
- **通知方式**：站内通知、邮件、浏览器推送（暂时忽略微信推送）
- **通知历史记录**：只保留6小时
- 通知频率设置

**5. 热点展示：**
- **过滤后的最终结果展示**：
  - 数量统计
  - 热度排序
  - 标题简述
  - 可跳转链接
  - 详情内容
  - **标明查询关键词**：直观区分哪条结果是按哪些关键词查询到的
  - **重要性分级**：紧急、高、中、低四个等级
- 按时间、热度、相关性排序
- 热点详情预览
- 收藏和分享功能
- 热点趋势分析

**6. 用户管理：**
- 注册、登录（支持第三方登录）
- 个人设置
- **权限管理**：每个登录用户只能使用自己的关键词和热点结果
- 操作历史记录

### 1.3 用户交互流程设计

**主要流程：**
1. **用户注册/登录** → 进入系统
2. **关键词管理** → 添加监控关键词 → 设置优先级 → 停/启用关键词 → 点击“立即扫描”（可选）
3. **定时设置** → 配置搜集频率和范围（包括新增的信息源）
4. **热点查看** → 浏览过滤后的热点结果 → 查看关键词标记和重要性等级
5. **通知管理** → 查看页面消息通知 → 处理热点
6. **个人设置** → 配置通知方式 → 调整系统设置

**关键交互：**
- 关键词添加：表单提交，支持批量添加
- 立即扫描：按钮点击后显示加载状态，完成后显示结果
- 热点预览：悬停预览或点击详情
- **页面消息通知**：顶部右边角的通知图标，点击后侧边栏展开，查看通知内容列表
- 数据可视化：热点趋势图表

### 1.4 数据结构定义

**2. 关键词表(Keyword)**
- id: 主键
- user_id: 外键（关联用户）
- keyword: 关键词内容
- priority: 优先级（1-3）
- status: 状态（启用/禁用）
- created_at: 创建时间
- updated_at: 更新时间

**3. 热点表(Hotspot)**
- id: 主键
- title: 标题
- content: 内容
- source: 来源
- url: 链接
- hot_score: 热度
- relevance: 相关度
- keyword_id: 外键（关联关键词）
- ai_score: AI评分
- importance: 重要性等级（紧急、高、中、低）
- is_filtered: 是否被过滤
- like_count: 点赞数
- view_count: 阅读量
- published_at: 发布时间
- created_at: 创建时间
- updated_at: 更新时间

**4. 通知表(Notification)**
- id: 主键
- user_id: 外键（关联用户）
- hotspot_id: 外键（关联热点）
- type: 通知类型
- status: 状态（已读/未读）
- created_at: 创建时间

**5. 搜集配置表(CollectionConfig)**
- id: 主键
- user_id: 外键（关联用户）
- frequency: 搜集频率
- sources: 搜集来源（包括新增的信息源）
- created_at: 创建时间
- updated_at: 更新时间

**6. 收藏表(Favorite)**
- id: 主键
- user_id: 外键（关联用户）
- hotspot_id: 外键（关联热点）
- created_at: 创建时间

### 1.5 通知方式设计

**支持的通知方式：**
1. **页面消息通知**：在页面上实时显示
2. **站内通知**：系统内实时提醒
3. **邮件通知**：发送邮件到用户邮箱
4. **浏览器推送**：Web浏览器通知
5. **微信推送**：暂时忽略

**通知触发条件：**
- 发现新热点
- 热点热度变化超过阈值
- 关键词相关度超过阈值
- 热点重要性等级为紧急或高

**通知历史记录：**
- 只保留6小时的通知历史

## 2. 技术实现方案

### 2.1 系统架构设计

**整体架构：**
- **前端层**：响应式Web应用
- **后端层**：RESTful API服务
- **数据层**：SQLite数据库
- **AI服务层**：OpenRouter AI服务
- **外部服务层**：搜索引擎和社交媒体API

**模块架构：**

```
┌─────────────────┐
│ 前端应用        │
│ (React + Vite + TailwindCSS) │
└────────┬────────┘
         │
┌────────▼────────┐
│ 后端API服务     │
│ (Node.js + Express) │
└────────┬────────┘
         │
┌────────▼────────┐     ┌─────────────────┐
│ 数据处理模块    │────►│ AI服务(OpenRouter) │
└────────┬────────┘     └─────────────────┘
         │
┌────────▼────────┐     ┌─────────────────┐
│ 定时任务模块    │────►│ 外部数据源     │
└────────┬────────┘     └─────────────────┘
         │
┌────────▼────────┐
│ 数据库          │
│ (SQLite)        │
└─────────────────┘
```

**数据流设计：**
1. **用户操作流程**：
   - 用户注册/登录 → 前端发送请求 → 后端验证 → 返回用户信息
   - 用户添加关键词 → 前端发送请求 → 后端存储 → 返回成功信息
   - **用户点击“立即扫描”** → 前端发送请求 → 后端读取用户关键词 → 调用外部API搜集数据 → AI处理 → 过滤结果 → 存储热点 → 发送通知

2. **定时任务流程**：
   - 定时任务触发 → 后端读取用户关键词 → 调用外部API搜集数据 → AI处理 → 过滤结果 → 存储热点 → 发送通知

3. **通知流程**：
   - 热点生成 → 后端判断通知条件 → 发送通知 → 前端接收并显示

**前端技术栈：**

**外部数据源：**
- **搜索引擎**：Google, Bing, DuckDuckGo

### 2.3 API接口设计

**1. 用户管理API：**
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户信息
- `PUT /api/auth/profile` - 更新用户信息

**2. 关键词管理API：**
- `GET /api/keywords` - 获取用户关键词列表
- `POST /api/keywords` - 添加关键词
- `PUT /api/keywords/:id` - 更新关键词
- `DELETE /api/keywords/:id` - 删除关键词
- `POST /api/keywords/scan` - 立即扫描所有关键词

**3. 热点监控API：**
- `GET /api/hotspots` - 获取热点列表（支持过滤和排序）
- `GET /api/hotspots/overview` - 获取热点概览数据
- `POST /api/hotspots/:id/favorite` - 收藏热点
- `DELETE /api/hotspots/:id/favorite` - 取消收藏

**4. 通知管理API：**
- `GET /api/notifications` - 获取通知列表
- `PUT /api/notifications/:id/read` - 标记通知为已读
- `DELETE /api/notifications` - 清除通知

**5. 配置管理API：**
- `GET /api/config` - 获取用户配置
- `PUT /api/config` - 更新用户配置

### 2.5 AI集成方案（OpenRouter服务对接）

**1. 集成架构：**
- **API调用层**：封装OpenRouter API调用
- **数据处理层**：处理输入数据和AI返回结果
- **业务逻辑层**：根据AI结果进行业务决策

**2. 对接流程：**
1. **准备数据**：收集到的热点内容
2. **调用OpenRouter API**：发送内容进行分析
3. **处理结果**：解析AI返回的分析结果
4. **应用过滤规则**：根据分析结果应用过滤规则
5. **重要性分级**：基于AI分析结果进行重要性分级

**3. API调用示例：**
```javascript
const analyzeContent = async (content, keyword) => {
  const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${process.env.OPENROUTER_API_KEY}`
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: [
        {
          role: 'system',
          content: `你是一个内容分析助手，需要分析以下内容与关键词"${keyword}"的相关性，并评估内容质量。请返回：
          1. 相关性评分（0-100）
          2. 内容质量评分（0-100）
          3. 是否为低质量内容
          4. 是否为重复内容
          5. 重要性等级（紧急、高、中、低）`
        },
        {
          role: 'user',
          content: content
        }
      ],
      temperature: 0.3
    })
  });
  return response.json();
};
```

**4. 过滤规则实现：**
- **内容质量**：AI评分低于60的内容过滤
- **时效性**：发布时间超过48小时的内容过滤
- **互动量**：点赞低于50或阅读量低于1500的内容过滤
- **来源可信度**：根据来源域名进行可信度评估
